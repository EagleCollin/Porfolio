D&D Character Storage

The Purpose

I started this application for one main reason: I didn't like being restricted to how many characters I could store at one time on another web application.
This led me down the rabbit hole of "How in depth do I want to make this application?" That discovery process brought me to a singular conclusion:
I want this application to simplify the character creation process for any wanna be Dungeons and Dragons nerd.


Functionality

The main sections of the application are the authorization, the api calls, and the actual display of the information.

Authorization

The authorization was setup to allow the user to save their information so they could save the information to come back to later.
This application uses Firebase as the database for storing users and their respective character data.

API Calls

The API used for this application only has the baseline Player's Handbook as content for searching and therefore only has the default classes
available to select from when selecting a character.

Data Display

The list of characters associated with a particular user is displayed by looping through the array that is retrieved from Firebase and then setting each anchor tag list item to the value of the character's name, class, and level. Additionally the character sheet display is made entirely with CSS and assigns values based on the selected character's ID which is retrieved from the URL.

Styling

The majority of the styling for the application, besides the character sheet display, is made with Angular Material and some slight changes made through personal CSS.
